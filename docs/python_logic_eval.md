# Python Logic Eval：從條款到可執行邏輯的演進路徑

## 1. 核心概述
`python_logic_eval` 是本專案中的關鍵欄位，旨在將保險條款中的「自然語言給付邏輯」轉譯為「結構化運算表達式」。它不只是文字摘要，而是具備運算潛力的邏輯描述。

## 2. 當前階段：概念驗證 (PoC)
目前的主要任務是進行「邏輯欄位化」的概念驗證：
- **目標**：將條款公式改寫為「各項參數欄位的組合計算」。
- **重點**：
    - 識別出邏輯中涉及的參數（如：`insured_amount`, `policy_year`, `age`）。
    - 建立參數間的初步數學關係（如：`insured_amount * 1.2`）。
    - 驗證 LLM 是文本中抽取否能準確從非結構化出正確的運算因子。

## 3. 未來應用：從描述到執行

### 3.1 引入 DSL 
未來計畫將 `python_logic_eval` 進一步正規化為一套專為保險業設計的 **DSL**：
- **安全執行**：避免直接執行原生 Python `eval()` 的安全性風險。
- **跨平台一致性**：DSL 可以被轉譯為 JavaScript (前端試算)、SQL (後端分析) 或 Python (核心引擎)。
- **語法特化**：支援保險特有的邏輯，如 `LOOKUP(表格, 維度)`、`RECURSIVE_SUM()` 等。

### 3.2 實務應用場景

#### A. 自動化理賠試算引擎
- **應用**：理賠系統直接載入該欄位邏輯。
- **效益**：理賠人員僅需輸入事實參數（如：住院天數），系統即可自動執行邏輯產出給付金額，大幅降低人工計算錯誤。

#### B. 精算模擬與產品壓力測試
- **應用**：利用 DSL 進行大規模模擬。
- **效益**：快速調整某參數（如：給付倍數），即時觀察對整體準備金或賠付率的影響。

#### C. 自動化測試與回溯驗證
- **應用**：將條款邏輯視為「測試預期結果」的來源。
- **效益**：當系統核心更新時，可自動比對系統計算結果是否仍符合 `python_logic_eval` 定義的原始邏輯。

## 4. 演進路徑
1. **[當前] 邏輯萃取**：從條款中提取出人類可讀、機器可理解的初步公式。
2. **[中期] 參數綁定**：將公式中的變數與 `standard_parameters` (資料白板) 完全勾稽。
3. **[長期] DSL 運算化**：正式切換為 DSL 語法，實現全自動化的邏輯執行。
